Algorithms

Recursion

A programming technique where a function calls itself within its own definition. It is usually used to solve problems that can be broken down into smaller instances of the same problem.

Dynamic Programming

Dynamic Programming is a method used to solve complex problems by breaking them down into simpler subproblems. By solving each subproblem only once and storing the results, it avoids redundant computations.

Greedy

Greedy Algorithm is a problem-solving approach that builds a solution step by step, always choosing the option that offers the most immediate benefit at each step.

Backtracking

Follow Recursion with the option to revert and traces back if the solution from current point is not feasible.

Divide and Conquer

A strategy to solve problems by dividing them into smaller subproblems, solving those subproblems, and combining the solutions to obtain the final solution.

Randomized algorithms

Are algorithms that use random choices to solve problems. They might give different answers each time, but they are often faster and easier than regular methods.

Algorithm Categories

Searching algorithms

Searching algorithms are used to locate specific data within a large set of data. It helps find a target value within the data.

Searching

Searching algorithms are used to locate specific data within a large set of data. It helps find a target value within the data. There are various types of searching algorithms, each with its own approach and efficiency.

Linear Search

Checks each item one by one until it finds the target. Simple but slow for large data.

Binary Search

Splits a sorted list in half repeatedly to find the target quickly. Very fast but requires sorted data.

Sorting algorithms

Sorting algorithms are used to arrange the elements of a list in a specific order, such as numerical or alphabetical. It organizes the items in a systematic way, making it easier to search for and access specific elements.

Sorting

Sorting algorithms are used to arrange the elements of a list in a specific order, such as numerical or alphabetical. It organizes the items in a systematic way, making it easier to search for and access specific elements.

Bubble Sort

Compares and swaps adjacent items repeatedly. Easy to understand but slow.

Merge Sort

Divides the list into halves, sorts them, and merges. Reliable and efficient.

Quick Sort

Picks a "pivot," splits the list around it, and sorts recursively. Fast for most cases.

Graph algorithms

Graph algorithms are used to solve problems related to graph data structures. They help find paths, cycles, and connectivity between nodes in a graph.

Time Complexity

How runtime increases as input size grows

  • O(1) - Constant time
  • O(log n) - Logarithmic time
  • O(n) - Linear time
  • O(n log n) - Linearithmic time
  • O(n²) - Quadratic time
  • O(2ⁿ) - Exponential time
