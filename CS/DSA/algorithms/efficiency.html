Algorithm that makes things fast:

Understanding algorithm efficiency is crucial for writing performant code. Learn about time and space complexity analysis and how to choose the right algorithm for your needs.

Time Complexity

Time complexity measures how runtime increases as input size grows. It answers the question: "How does the algorithm scale with larger input?"

  • O(1) - Constant Time

    Runtime is the same regardless of input size. Examples: Accessing an array element, adding an item to a stack.

  • O(log n) - Logarithmic Time

    Runtime grows logarithmically with input size. Examples: Binary search, balanced tree operations.

  • O(n) - Linear Time

    Runtime grows linearly with input size. Examples: Linear search, traversing an array.

  • O(n log n) - Linearithmic Time

    Common in efficient sorting algorithms. Examples: Merge sort, heap sort, quick sort (average case).

  • O(n²) - Quadratic Time

    Runtime grows with the square of input size. Examples: Bubble sort, insertion sort, simple nested loops.

  • O(2ⁿ) - Exponential Time

    Runtime doubles with each addition to input size. Examples: Recursive calculation of Fibonacci numbers, traveling salesman problem.

Space Complexity

Space complexity measures how memory usage increases with input size. It answers the question: "How much additional memory does the algorithm need?"

  • O(1) - Constant Space

    Uses a fixed amount of memory regardless of input size. Examples: Iterative algorithms that use a few variables.

  • O(log n) - Logarithmic Space

    Memory usage grows logarithmically with input size. Example: Recursive binary search.

  • O(n) - Linear Space

    Memory usage grows linearly with input size. Examples: Creating a new array to store results.

  • O(n²) - Quadratic Space

    Memory usage grows with the square of input size. Example: Creating a 2D matrix for all pairs of items.

Algorithm Efficiency Examples

Searching

Linear Search: O(n) time

Binary Search: O(log n) time

Binary search is significantly faster for large datasets, but requires sorted input.

Sorting

Bubble Sort: O(n²) time

Merge Sort: O(n log n) time

For large datasets, merge sort outperforms bubble sort by orders of magnitude.

Data Structures

Array Access: O(1) time

Linked List Access: O(n) time

Hash Table Lookup: O(1) average time

Choosing the right data structure is crucial for algorithm efficiency.
