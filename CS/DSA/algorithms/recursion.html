Recursion

A programming technique where a function calls itself within its own definition. It is usually used to solve problems that can be broken down into smaller instances of the same problem.

What is Recursion?

Recursion is a method of solving a problem where the solution depends on solutions to smaller instances of the same problem. In programming, this takes the form of a function that calls itself with a modified parameter until it reaches a base case.

A recursive function needs two main components:

  • Base Case: A condition that stops the recursion to prevent infinite loops
  • Recursive Case: Where the function calls itself with a changed input to approach the base case

Recursion is particularly useful for tasks that can be defined in terms of similar subtasks, especially when the solution is easier to express in recursive terms than using loops.

Understanding Recursion Through Examples

Classic Example: Calculating Factorial

The factorial of a non-negative integer n, denoted n!, is the product of all positive integers less than or equal to n.

For example: 5! = 5 × 4 × 3 × 2 × 1 = 120

We can define this recursively as: n! = n × (n-1)! with base case 0! = 1

Fibonacci Sequence

The Fibonacci sequence is defined as:

  • F(0) = 0 (base case)
  • F(1) = 1 (base case)
  • F(n) = F(n-1) + F(n-2) (for n > 1)

This naturally lends itself to a recursive solution.

// Example 4: Binary search using recursion
int binarySearch(int arr[], int left, int right, int x) {
    if (right >= left) {
        int mid = left + (right - left) / 2;
        
        // If the element is present at the middle
        if (arr[mid] == x)
            return mid;
        
        // If element is smaller than mid, search in left subarray
        if (arr[mid] > x)
            return binarySearch(arr, left, mid - 1, x);
        
        // Else search in right subarray
        return binarySearch(arr, mid + 1, right, x);
    }
    
    // Element is not present in array
    return -1;
}

import os

def list_files_recursively(directory):
    files = []
    
    # List all entries in the directory
    entries = os.listdir(directory)
    
    for entry in entries:
        full_path = os.path.join(directory, entry)
        
        # If the entry is a file, add it to the list
        if os.path.isfile(full_path):
            files.append(full_path)
        
        # If the entry is a directory, recursively list its files
        elif os.path.isdir(full_path):
            files.extend(list_files_recursively(full_path))
    
    return files

// Example 5: Deep copy of an object
function deepCopy(obj) {
    // Base cases: null or primitive values
    if (obj === null || typeof obj !== 'object')
        return obj;
    
    // Handle arrays
    if (Array.isArray(obj)) {
        return obj.map(item => deepCopy(item));
    }
    
    // Handle objects
    const copy = {};
    for (const key in obj) {
        if (Object.prototype.hasOwnProperty.call(obj, key)) {
            copy[key] = deepCopy(obj[key]);
        }
    }
    return copy;
}

// Example 6: Flattening nested arrays
function flattenArray(arr) {
    let result = [];
    
    for (let i = 0; i < arr.length; i++) {
        if (Array.isArray(arr[i])) {
            // Recursively flatten nested arrays and concatenate
            result = result.concat(flattenArray(arr[i]));
        } else {
            // Add non-array element to result
            result.push(arr[i]);
        }
    }
    
    return result;
}

// Example 7: DOM traversal
function countElementsInDOM(element) {
    let count = 1; // Count the current element
    
    // Base case: element has no children
    if (element.children.length === 0) {
        return count;
    }
    
    // Recursive case: add counts from all children
    for (let i = 0; i < element.children.length; i++) {
        count += countElementsInDOM(element.children[i]);
    }
    
    return count;
}

function listFiles($dir) {
    $files = array();
    
    // Get all entries in the directory
    $entries = scandir($dir);
    
    foreach ($entries as $entry) {
        // Skip . and .. directories
        if ($entry === '.' || $entry === '..') {
            continue;
        }
        
        $path = $dir . '/' . $entry;
        
        // If it's a file, add it to the list
        if (is_file($path)) {
            $files[] = $path;
        } 
        // If it's a directory, recursively list files inside it
        elseif (is_dir($path)) {
            $files = array_merge($files, listFiles($path));
        }
    }
    
    return $files;
}

        if (is_array($value)) {
            // If the element is an array, recursively sum its elements
            $sum += nestedArraySum($value);

    }

Recursion vs Iteration

While recursion can make code more elegant and readable for certain problems, there are trade-offs to consider:

Recursion Iteration
Memory Usage Higher (maintains function call stack) Lower (no additional function calls)
Speed Usually slower (function call overhead) Typically faster
Readability Often cleaner for certain problems Can be more complex for problems that are naturally recursive
Risk Stack overflow for deep recursion Infinite loops if termination condition is wrong

Many problems can be solved using either approach, and the choice often depends on the specific requirements of the situation.

Common Applications of Recursion

  • Tree and graph algorithms: Traversal, searching, and manipulation
  • Divide and conquer algorithms: Merge sort, quicksort
  • Backtracking algorithms: N-Queens problem, Sudoku solver
  • Dynamic programming: Many DP problems start with a recursive solution
  • Fractals: Generating self-similar patterns
  • Parsing expressions: Evaluating arithmetic expressions

Tips for Writing Recursive Functions

  1. Always identify the base case(s) first to prevent infinite recursion
  2. Make sure the recursive calls move toward the base case
  3. Consider using memoization to avoid redundant calculations in recursive functions
  4. Be mindful of the stack size limit in your programming language
  5. For complex problems, draw out the recursive calls to understand the flow
  6. Consider if tail recursion is applicable and if your language optimizes for it
