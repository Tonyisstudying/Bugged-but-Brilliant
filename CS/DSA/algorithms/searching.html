Searching Algorithms

Searching algorithms are essential tools in computer science used to locate specific items within a collection of data. In this tutorial, we are mainly going to focus upon searching in an array. When we search an item in an array, there are two most common algorithms used based on the type of input array.

Linear Search

It is used for an unsorted array. It mainly does one by one comparison of the item to be search with array elements. It takes linear or O(n) Time.

For example: Consider the array arr[] = {10, 50, 30, 70, 80, 20, 90, 40} and key = 30

#include int search(int arr[], int n, int x) { // Iterate over the array in order to // find the key x for (int i = 0; i < n; i++) if (arr[i] == x) return i; return -1; } // Driver code int main(void) { int arr[] = { 2, 3, 4, 10, 40 }; int x = 10; int n = sizeof(arr) / sizeof(arr[0]); // Function call int result = search(arr, n, x); (result == -1) ? printf("Element is not present in array") : printf("Element is present at index %d", result); return 0; }

Time Complexity: O(n) - In the worst case, we need to check all n elements.

Space Complexity: O(1) - Uses a constant amount of extra space.

When to use: Small arrays or unsorted data where setup costs of more advanced algorithms isn't justified.

Binary Search

Splits a sorted list in half repeatedly to find the target quickly. Very fast but requires sorted data.

#include int search(int arr[], int n, int x) { // Iterate over the array in order to // find the key x for (int i = 0; i < n; i++) if (arr[i] == x) return i; return -1; } // Driver code int main(void) { int arr[] = { 2, 3, 4, 10, 40 }; int x = 10; int n = sizeof(arr) / sizeof(arr[0]); // Function call int result = search(arr, n, x); (result == -1) ? printf("Element is not present in array") : printf("Element is present at index %d", result); return 0; } return -1; // Target not found

Time Complexity: O(log n) - The search space is halved with each iteration.

Space Complexity: O(1) - Uses a constant amount of extra space.

When to use: Large sorted arrays where lookup speed is critical.
