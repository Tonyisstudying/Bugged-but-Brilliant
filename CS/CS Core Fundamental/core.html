Computer Science Mindmap

Computer Science Core Fundamentals

Hardware: is the physical part of the computer.

  • Input Devices: a keyboard, mouse, or camera, let us put data into the system.
  • Output Devices: monitors, speakers, and printers, show the results back to us.
  • Storage Devices
    RAM: fast but temporary
    ROM: permanent storage
    Larger Devices: HDD, SSD
  • CPU: the brain of the computer, which calculates, makes decisions, and controls how data moves.
  • Motherboard: connects all parts together, and the power supply gives the computer electricity.

Software: the set of programs that make the hardware useful.

  • Operating System: is very important because it manages all the basic work of the computer. It handles memory, files, devices, and running programs. It also gives us a way to interact with the computer, either through text commands or easy-to-use icons and windows.
  • Application Software: helps users perform specific tasks, like word processing or web browsing.
  • Programming Software: provides tools for developers to create software applications.

Internet: connects computers worldwide.

The World Wide Web (WWW) is where websites and online resources live. We use domain names (like google.com) instead of numbers (IP addresses), and browsers (like Chrome or Safari) let us visit sites. Search engines (like Google) help us find information.

Computer Networks

1. Fundamentals

A computer network is a system of interconnected devices that communicate and share resources using standard protocols. Its goals include enabling communication, sharing resources like files and printers, ensuring reliability through redundancy, supporting scalability, and optimizing efficiency. Networks consist of end devices (PCs, laptops, smartphones, servers), networking devices (routers, switches, hubs, access points, firewalls), and transmission media (wired or wireless). Communication is governed by protocols such as TCP/IP, HTTP, DNS, and FTP. Key models include OSI (7 layers) and TCP/IP (4 layers), which define functions from physical transmission to application-level communication. Networks vary in scale from PAN, LAN, MAN, WAN to enterprise networks.

2. Subfields of Computer Networks

  • 2.1 Network Architecture & Protocols: Defines network design (centralized vs decentralized) and communication rules, including TCP/IP, UDP, HTTP/HTTPS, FTP, email protocols, DNS, and routing protocols like BGP, OSPF, and RIP.
  • 2.2 Routing & Switching: Switching operates on MAC addresses (Layer 2) and includes Ethernet, VLAN, STP; routing operates on IP addresses (Layer 3), finding optimal paths through static or dynamic protocols. NAT translates private to public IPs.
  • 2.3 Wireless & Mobile Networking: Covers Wi-Fi, mobile networks (3G/4G/5G), Bluetooth, NFC, ad hoc and mesh networks, and mobility management via Mobile IP.
  • 2.4 Network Security: Addresses malware, DoS/DDoS, MITM, spoofing, and phishing using firewalls, IDS/IPS, VPNs, encryption, and zero-trust principles.
  • 2.5 Cloud & Data Center Networking: Focuses on large-scale infrastructures, SDN/NFV for virtualization, virtual networking with VMs/containers, and load balancing.
  • 2.6 IoT Networking: Connects everyday devices (sensors, cameras, appliances) using lightweight protocols (MQTT, CoAP) and low-power wireless technologies (ZigBee, Z-Wave, LoRaWAN), with challenges in scalability and security.
  • 2.7 Performance & QoS: Measured by bandwidth, latency, jitter, and packet loss; optimized through congestion control, traffic shaping, scheduling, and service differentiation.
  • 2.8 Network Management & Monitoring: Involves configuration, fault detection, performance measurement, and security auditing using SNMP, traffic analysis tools, monitoring software, and automation frameworks.
  • 2.9 Emerging Areas: Includes 6G networking, quantum networking, satellite networks, AI-driven networking, and edge/fog computing.

1. Foundations

  • Protecting information systems from unauthorized access, modification, or disruption.
  • Guided by the CIA triad: confidentiality, integrity, availability.
  • Privacy: control over how personal data is collected, used, and shared.
  • Privacy by design: embedding protections from the ground up.

2. Core Areas of Security

  • Cryptography: symmetric/asymmetric encryption, hash functions, digital signatures, zero-knowledge proofs, homomorphic encryption.
  • Network Security: firewalls, VPNs, TLS/SSL, defenses against DoS/MITM.
  • System Security: OS hardening, patching, access control, malware defense.
  • Application Security: secure coding, vulnerability testing, exploit prevention.
  • Cybersecurity Operations: monitoring, threat intelligence, incident response.

3. Core Areas of Privacy

  • Data Privacy: anonymization, minimization, GDPR, CCPA compliance.
  • Differential Privacy: analyze data without leaking individual info.
  • Privacy-Preserving Computation: homomorphic encryption, multi-party computation, federated learning.
  • Identity & Anonymity: Tor, zero-knowledge credentials, decentralized identity systems.
